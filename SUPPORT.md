# Support

Thank you for using FaultPlane.

This document explains where to find documentation, ask questions, report issues, and request help.

Before opening a support request, please check the available documentation and existing discussions.

---

- `ROADMAP.md` — Development direction and planned milestones

```
README.md

ARCHITECTURE.md

Documentation Guides

      ↓

Issues / Discussions
```

# Community Questions

For general questions, discussions, and design conversations:

Use:

- GitHub Discussions
- GitHub Issues with the Question template

Good discussion topics include:

- architecture ideas
- deployment questions
- integration approaches
- development guidance
- feature discussions

Please search existing discussions before creating a new topic.

# Bug Reports

If you discover a bug, please create a GitHub Issue.

A useful bug report should include:

## Environment Information

- FaultPlane version or commit hash
- Go version
- Operating system
- Deployment environment
- Relevant configuration details

---

## Problem Description

Include:

- expected behavior
- actual behavior
- reproduction steps
- error messages
- relevant logs

Example:

```
FaultPlane Version:
Go Version:
Operating System:

Steps to Reproduce:

1.
2.
3.

Expected Result:

Actual Result:

Logs:
```

Clear reproduction steps help maintainers investigate issues faster.

---

# Feature Requests

Feature requests are welcome.

Before proposing a feature, consider:

- Does it solve a common infrastructure problem?
- Does it align with FaultPlane's goals?
- Can it remain simple and maintainable?

Feature discussions should include:

- problem description
- proposed solution
- possible alternatives
- expected impact

---

# Security Issues

Do **not** report security vulnerabilities through public GitHub issues.

For security-related concerns:

Please follow:

```
SECURITY.md
```

Security reports should include:

- affected version
- vulnerability description
- reproduction steps
- potential impact
- supporting evidence

Responsible disclosure helps protect users and maintain project security.

---

# Response Expectations

FaultPlane is maintained by an open-source community.

Response times may vary depending on:

- issue complexity
- maintainer availability
- project activity

Critical issues will receive priority.

- clear
